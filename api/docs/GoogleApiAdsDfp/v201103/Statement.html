Class Statement

Description
Description | Vars (details) | Methods (details)

Captures the {@code WHERE}, {@code ORDER BY} and {@code LIMIT} clauses of a PQL query. Statements are typically used to retrieve objects of a predefined domain type, which makes SELECT clause unnecessary.

An example query text might be {@code "WHERE status = 'ACTIVE' ORDER BY id LIMIT 30"}.

Statements also support bind variables. These are substitutes for literals and can be thought of as input parameters to a PQL query.

An example of such a query might be {@code "WHERE id = :idValue"}.

If using an API version newer than V201010, the value for the variable idValue must then be set with an object of type Value and is one of NumberValue, TextValue or BooleanValue. If using an API version older than or equal to V201010, the value for the variable idValue must then be set with an object of type Param and is one of DoubleParam, LongParam or StringParam.
